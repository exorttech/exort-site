# Карта перехода на новую админ-панель Exort

Новая панель сохраняет визуальный слой `admin-showcase`, но использует существующую рабочую авторизацию, серверные действия и модели данных `demo-admin`. Поддельные массивы и локальные генераторы демо-данных удалены вместе с `js/admin-showcase.js`.

| Раздел или действие | Интерфейс | Источник данных / API | Статус |
| --- | --- | --- | --- |
| Вход по PIN | Новый экран входа | `login` в `/api/exort-admin`, временная подписанная сессия в `sessionStorage` | Работает |
| Загрузка заведения | Шапка, меню, настройки | Публичное чтение активного `restaurants`, после входа — `getData` | Работает |
| Обзор | Метрики и проблемы меню | `getData` + реальный `getAnalytics` за сегодня | Работает |
| Блюда | Список, поиск, фильтры, редактор | `getData`, `saveItem`, `deleteItem`, `toggleStock` | Работает |
| Фото блюд | Загрузка и замена изображения | `uploadItemPhoto` | Работает |
| Категории | Список, создание, редактирование, сортировка | `saveCategory`, `sortCategories` | Работает; удаления категории в API нет |
| Стоп-лист | Быстрое включение и выключение | `toggleStock` | Работает |
| Переводы RU / KZ / EN | Поля редактора, фильтры отсутствующих переводов | Ручной ввод, `translate`, `translateMissing` | Работает при настроенном сервисе перевода |
| Аналитика | Сегодня, 7 дней, 30 дней, всё время | `getAnalytics`, таблица `menu_analytics_events` | Работает на реальных событиях |
| QR-коды | Источники, создание, изменение, архив | `getQrSources`, `createQrSource`, `updateQrSource`, `archiveQrSource` | Работает |
| Настройки заведения | Реальные текущие имя и ссылка меню | Данные `getData` | Только чтение: серверного действия сохранения нет |
| Интеграции | r_keeper и iiko | Сервер не возвращает состояние интеграций | Честно отмечены как не подключённые; фиктивных статусов нет |

## Ограничения текущего backend

- Нет серверных действий для изменения профиля заведения и удаления категории.
- Нет модели или endpoint для статусов r_keeper/iiko.
- Автоперевод зависит от отдельного настроенного сервиса; ручное редактирование доступно всегда.
- Выборка аналитики ограничена существующим серверным пределом в 20 000 событий.
- Текущая модель контента поддерживает RU, KZ и EN; новый интерфейс не обещает неподдерживаемый четвёртый язык.
